Introdução ao SASS

SASS (Syntactically Awesome Stylesheets) é uma linguagem de extensão CSS que adiciona poder e elegância ao CSS básico. Criado por Hampton Catlin em 2006, SASS tem evoluído continuamente, sendo mantido por Natalie Weizenbaum e Chris Eppstein. A principal motivação por trás do SASS é melhorar a escrita de CSS, tornando-o mais eficiente, modular e reutilizável.

Filosofia e Objetivos

  • Modularidade:

    Facilitar a divisão de grandes folhas de estilo em pequenos módulos.

  • Reutilização:

    Permitir a reutilização de estilos através de mixins e funções.

  • Manutenibilidade:

    Melhorar a manutenção de grandes projetos CSS.

  • Flexibilidade:

    Oferecer ferramentas avançadas para criar folhas de estilo dinâmicas e complexas.

Componentes do SASS

  • Variáveis:

    Permitem armazenar valores que podem ser reutilizados em toda a folha de estilo.

    $primary-color: #333;

body {
  color: $primary-color;
}

  • Aninhamento:

    Permite escrever CSS de forma hierárquica, imitando a estrutura HTML.

    nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }

  li { display: inline-block; }

  a {
    text-decoration: none;
    color: $primary-color;
  }
}

  • Partials e Imports:

    Facilitam a organização do código CSS em vários arquivos.

    // _base.scss
body {
  font-family: Arial, sans-serif;
}

// main.scss
@import 'base';

  • Mixins:

    Permitem criar blocos de código CSS reutilizáveis.

    @mixin border-radius($radius) {
  -webkit-border-radius: $radius;
  -moz-border-radius: $radius;
  border-radius: $radius;
}

.box { @include border-radius(10px); }

  • Funções:

    Similar às funções em linguagens de programação, retornam valores com base em parâmetros.

    @function px-to-rem($px, $base: 16) {
  @return #{$px / $base}rem;
}

body {
  font-size: px-to-rem(16);
}

Como Funciona o SASS

Processo de Compilação

  1. Linhagem de Comando:

    Compilação via linha de comando usando o comando sass.

    sass source/styles.scss build/styles.css

  2. Task Runners:

    Utilização de ferramentas como Gulp para compilar automaticamente o SASS.

    // Gulpfile.js
const gulp = require('gulp');
const sass = require('gulp-sass')(require('sass'));

gulp.task('sass', function () {
  return gulp.src('source/styles.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(gulp.dest('build'));
});

  3. IDEs:

    Integração com IDEs como Visual Studio Code para compilar SASS diretamente no ambiente de desenvolvimento.

Estrutura do Código

O SASS pode ser escrito em duas sintaxes: .scss e .sass. A sintaxe .scss é mais semelhante ao CSS tradicional, enquanto a .sass usa indentação para definir a hierarquia.

  • Exemplo em .scss:

    $font-stack: Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}

Fundamentos Teóricos

Introdução

LESS (Leaner Style Sheets) é uma linguagem de extensão CSS que adiciona funcionalidades como variáveis, mixins, funções e aninhamento ao CSS, tornando-o mais dinâmico e fácil de manter. Criado por Alexis Sellier em 2009, LESS foi um dos primeiros preprocessadores CSS amplamente adotados e inspirou o desenvolvimento de outras ferramentas similares, como SASS.

Filosofia e Objetivos

LESS foi desenvolvido com o objetivo de tornar o desenvolvimento de CSS mais eficiente e modular. Ele permite que os desenvolvedores escrevam código CSS de forma mais concisa e reutilizável, promovendo melhores práticas de programação e manutenção de grandes projetos de estilos. Seus principais objetivos incluem:

  • Simplificação: Facilitar a escrita e a leitura de CSS.
  • Modularidade: Permitir a divisão do código CSS em pequenos módulos reutilizáveis.
  • Manutenibilidade: Melhorar a organização e manutenção de projetos CSS complexos.
  • Compatibilidade: Ser compatível com o CSS tradicional, de modo que todo CSS válido também seja um código LESS válido.

Stylus

Stylus é uma linguagem de extensão para CSS, conhecida como preprocessador CSS, que adiciona funcionalidades avançadas para facilitar a escrita e manutenção de folhas de estilo. Criado por TJ Holowaychuk, Stylus oferece uma sintaxe flexível e uma variedade de recursos poderosos que tornam o CSS mais dinâmico e eficiente. A seguir, um detalhamento sobre Stylus:

Fundamentos do Stylus

Introdução

Stylus é um preprocessador CSS que permite escrever CSS de maneira mais eficiente e organizada. Ele suporta variáveis, mixins, aninhamento, funções, e operações, proporcionando uma sintaxe mais limpa e funcionalidades avançadas que simplificam a gestão de grandes projetos de estilos.

Filosofia e Objetivos

Stylus foi desenvolvido com os seguintes objetivos em mente:

  • Flexibilidade: Oferecer uma sintaxe extremamente permissiva e poderosa.
  • Produtividade: Facilitar a escrita de CSS e aumentar a produtividade dos desenvolvedores.
  • Modularidade: Permitir a divisão do código CSS em pequenos módulos reutilizáveis.
  • Manutenibilidade: Melhorar a organização e manutenção de projetos CSS complexos.
